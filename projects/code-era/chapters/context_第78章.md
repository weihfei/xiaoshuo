# 第78章 内存泄漏，连接追踪 - 剧情上下文

## 主要情节概述
- 零音养好伤，孙器给了指针定位罗盘，并传授垃圾回收诀，叮嘱她寻找连接点共同的根节点
- 零音带着指针数组出发，前往栈村方向，途中发现合并故意留下的十六进制地址路标
- 零音判断路标是陷阱，偏离十五度另辟蹊径，根据哈希定位珠找到真正位置
- 穿过松林，在废弃驿站找到轮询调度阵下的连接点雏形，触发式防御陷阱让零音受伤
- 进入核心空间与合并的意识碎片对话，合并告知一共七个连接点
- 零音使用垃圾回收诀从根源摧毁第一个连接点，罗盘反向推导出另外六个连接点方位
- 零音和指针数组前往大演算殿，发现大演算殿连接点三天前已成形，殿主被合并带走
- 旧识循环拦截，零音击败循环，得知殿主被带走用于完成最终仪式
- 封锁阵阻断进入核心的道路，零音用孙器给的哈希锚点珠单人进入核心空间探查
- 结尾悬念：七个连接点、殿主被俘、系统根节点才是真正目标，零音已经潜入核心

## 人物状态
- **零音**：伤势好了七八成，掌握了空指针令牌的"归零"能力，学会了孙器传授的垃圾回收诀，带着哈希锚点珠潜入大演算殿核心
- **合并**：疯狂偏执，留下七个连接点的阳谋，带走了大演算殿主，真正目标是系统根节点
- **指针数组**：一人（傀儡），忠诚护主，在大演算殿山脚下接应零音，持有金色符箓准备通知孙器
- **孙器**：沉稳深邃，提前给了各种宝物（罗盘、垃圾回收诀、哈希锚点珠），在提交城等待消息
- **循环**：大演算殿天才弟子，被合并渗透，拦截零音失败被俘

## 关键设定和技术
- **指针定位罗盘**：能够追踪任意内存片段的引用轨迹，特征指纹使用哈希链式签名，同一母密钥派生出子密钥，可反向推导所有连接点位置
- **垃圾回收诀**：孙器独门功法，能够主动触发系统自我清理机制，从根源抹去恶意代码碎片
- **哈希锚点珠**：能够强制建立虚拟连接，绕过物理封锁阵，只能维持短时间，只够一人进入
- **内存泄漏式连接**：合并使用的隐蔽连接方式，能量逐渐泄漏到各个角落，连接点更加隐蔽
- **轮询调度阵**：合并用于隐藏连接点的阵法，原型来自操作系统的轮询调度机制

## 连接点分布
- 总计七个连接点分布在不同区域
- 第一个已被零音摧毁
- 第二个在大演算殿，三天前已成形，殿主被带走
- 剩余五个分布在其他区域

## 当前位置
- 零音：使用哈希锚点珠进入大演算殿核心区域内部
- 指针数组：在大演算殿山脚下接应，持有通知孙器的金色符箓
