# 第42章 调试关卡 快排BUG

我的手放在石门上，轻轻一推。

石门没有发出想象中沉重的摩擦声，反而像是被润滑油浸润过一样，无声无息地向内打开，一股比第一层更加古老、更加干燥的气息扑面而来。

我们四个对视一眼，依次走了进去。

进门之后我才发现，第二层比第一层要大得多，足足有一个足球场那么大，穹顶更高，空气中漂浮着淡淡的荧光微粒，比第一层的晶矿灯光线更加柔和。

正中央是一个足足有五丈见方的巨大青石平台，平台表面刻满了密密麻麻的细小铭文，不用看我也知道，那肯定就是源代码了。

和第一层一样，正对着门口的墙壁上，同样刻着几个巨大的古字：

**创造者神殿，第二层：调试试炼**

下面依旧是一行小字试炼规则，我凑过去仔细读：

> 能编译，不一定能调试。写得出代码，不一定找得出BUG。
> 
> 中央石台上，刻着上古神兵「快速排序剑」的完整源代码。此剑能运行，能出剑，但是**有时候准，有时候不准**，胜败看运气。
> 
> 找出问题，修复BUG，让它永远稳定输出，就算你通过。
> 
> 找不到？或者修错了？代码错乱，能量爆炸，你们四个人，就一起留在这里，给神殿当能源吧。

读完规则，九尾忍不住"噗嗤"一声笑了出来，凑到我身边，用胳膊肘轻轻碰了碰我：

"万相大哥，我说什么来着，这创造者绝对是知道你要来，特意给你安排的对口专业啊！编译之后就是调试，这不就是你们程序员每天的日常工作吗？人家写BUG你调试，一天debug八个小时，闭着眼都能找出来吧？"

我也笑了，确实，说到debug找BUG，这还真就是我的日常工作。当了六年程序员，什么奇奇怪怪的BUG没见过？从生产环境偶现的Heisenbug，到依赖冲突导致的链接错误，再到多线程竞态条件，什么妖魔鬼怪都见过。

不就是一段有时候对有时候错的代码吗？我还真不信能难住我。

"零音，麻烦你检查一下，看看整个大厅有没有陷阱，有没有熵魔感染的痕迹。"我转过身，对零音说。

零音点点头，没有说话，只是闭上眼，全身散发出一层淡淡的银色光芒，那是她杀毒者的本源力量在扩散，扫描整个空间。几秒钟后，她睁开眼：

"没有活的陷阱，熵魔痕迹很淡，几乎可以忽略不计，应该是几百年前留下的死残留，不影响现在。安全。"

"清语，你要不要先找个地方休息一下？"我看向叶清语，她一路跟着我们进来，虽然没怎么动手，但也是一直绷紧着神经。

叶清语摇摇头，走到我身边，从储物袋里拿出一个水囊，纤长的手指轻轻拧开盖子，递到我面前，指尖不经意碰到我的手背，温温软软的，像一片花瓣轻轻拂过，让我心里微微一动。

"你才是最累的，刚才跟凌云天傀儡打了那么久，又吸收了记忆水晶，先喝口水缓一缓，不着急。"她柔声说，眉眼弯弯，眼神里满是关心，"反正都走到这儿了，不急这一时半刻，慢慢来，总能找到的。"

我接过水囊，仰起头喝了一口，甘甜的清水带着一丝淡淡的清心草香气，顺着喉咙滑下去，紧绷的神经一下子就放松了不少。我把水囊递回去，指尖又不小心碰到她的指尖，两人都微微顿了一下，然后她抿嘴微微一笑，收了回去，耳尖悄悄染上一点淡红。

这种含蓄的温柔，总是让我心里暖暖的。在这个陌生的世界，她是第一个对我好的人，这种感觉，跟九尾的俏皮、零音的冷艳都不一样。

"那就开始吧。"九尾活动了一下手腕，红色劲装勾勒出灵活的曲线，"我先上，凭我们狐族的直觉，哪里代码不稳定，我一摸就知道，先给你缩小范围。"

她说完，纵身跳上青石平台，两只手轻轻放在平台表面的铭文上，闭上眼睛，神念慢慢沉入代码中。

我们三个在台下看着，没过几分钟，九尾就睁开眼睛，指着平台上三个不同的区域：

"万相大哥，你看这三个地方，代码波动特别明显，比别的地方乱多了，肯定有问题。"

我纵身跳上去，蹲下来，顺着她指的地方看过去。

果然，这三段代码确实写得有点问题。

第一个地方：变量命名太随意，用了一个单字母`x`，根本不知道这个变量是干什么的，换了任何人来维护都看不懂。

第二个地方：注释写错了，注释说"这里选择基准元"，但实际代码做的是交换元素，文不对题。

第三个地方：有一行代码多余，计算了一个变量但是从来没用过，属于死代码。

我看完，忍不住笑了："九尾说的没错，这三个地方确实是问题，但都是不影响运行的小问题，属于代码坏味道，不是导致崩溃的根本BUG。"

"啊？"九尾有点惊讶，"那为什么波动这么大？"

"代码写得烂，当然波动大。"我站起身，拍了拍手上的灰，"就好比一件衣服，针脚缝得歪歪扭扭，但是不影响穿，只是不好看而已。真正穿的时候一洗就破，那是布料本身有问题，不是针脚的问题。"

我蹲下身，开始从上往下，逐行阅读这段快速排序剑的源代码。

说实话，算法本身写得真不错。快速排序的核心逻辑非常清晰：选择基准元，分割成左右两部分，递归排序。整个逻辑严丝合缝，partition函数写得很干净，没有问题。

我从头看到尾，核心逻辑真没毛病。

这就奇怪了。

逻辑没错，为什么会有时候对有时候错？

我摸着下巴，陷入思考。这种偶现BUG，是最头疼的，也是最考验调试能力的。如果每次都错，那反而好找，就怕这种时好时坏的。

"会不会是算力不够？"叶清语在下面问道，"凌云天当年算力不够，所以有时候运行不起来？"

"应该不是。"零音摇摇头，"如果是算力不够，那应该是每次都运行不起来，不会有时候行有时候不行。"

我也同意零音的判断。算力不够是系统性问题，不会分情况出问题。而且凌云天那时候已经是内核三级高手了，算力肯定够运行一段排序算法。

"万维那帮家伙要是现在进来，估计直接懵了。"九尾笑着说，"他们那帮闭源垄断的，只会用别人写好的API，自己从来没调试过，估计在这里卡个十年都找不到问题。"

说到万维联盟，我心里动了一下。他们现在估计还在第一层入口外面打转吧？没有钥匙，连第一层都进不来，只能在黑风峡谷里到处搜我们，估计搜得头都大了。

等我们拿到README出去，就是跟他们算帐的时候了。欧阳彻那小子带人伏击我们，这笔帐我可记着呢。

不想这些，回到BUG本身。

快速排序，逻辑对，有时候出问题……我脑子里过了一遍快速排序的各种经典坑。

基准元选择？啊！对了！

我脑子里灵光一闪，瞬间就想到了最经典的那个坑。干了六年程序员，什么BUG没见过，这种经典坑闭着眼都能背出来。

我立刻翻到代码里面选择基准元的部分，仔细一看。

得，还真是这个问题。我忍不住在心里吐槽：这都什么年代了，还写固定选最右基准元的快排，这代码要是放去我们公司code review，直接打回去重写，连给你改的欲望都没有。

代码写得非常简单直接：

```
pivot = arr[right]
```

永远选择最右边的元素作为基准元。

我忍不住笑了，得，问题找到了。

难怪有时候对有时候错，问题根本就不出在核心逻辑上，出在基准元选择上啊！

"我找到问题了。"我站起身，对台下三个人说。

"这么快？"九尾眼睛一下子就亮了，"哪里哪里？什么问题？"

"问题说出来其实很简单。"我指着选择基准元那一行代码，"你们看这里，它永远选最右边那个元素当基准元。这样会发生什么？"

三个人都摇头，她们不懂算法，当然不知道。

"如果数组本身就是无序的，那其实没什么问题，平均性能还是O(nlogn)。"我解释道，"但是如果数组本身就是有序的呢？比如从小到大排好了，你每次都选最右，那每次分割都会分成0和n-1，这样递归深度就会变成n，时间复杂度直接从O(nlogn)退化到O(n²)，而且递归栈太深会直接栈溢出，程序崩溃。"

我顿了顿，继续说："还有一种情况，如果数组里面有大量重复元素，每次分割都会极度不平衡，一边特别大一边特别小，性能也会暴跌，甚至出错。"

"所以……"叶清语若有所思，"就是说，如果遇到有序数组或者大量重复元素，它就会出问题；如果是无序的，它就没问题？所以才会有时候对有时候错，找不到规律？"

"没错！就是这样！"我点点头，佩服她理解得真快，"这就是根本原因。上古那位写代码的剑士，只想到了算法本身的逻辑，没想到工程上的边界情况。凌云天当年调试了几十年都找不到规律，那不是他笨，是他根本不知道还有这么一说，他只会悟道，不会工程调试啊。"

这就是降维打击。上古剑士拼了命悟了几百年都找不到的问题，在我这个现代程序员眼里，这就是入门级的经典BUG，教科书上都写得明明白白。

"那怎么修？"零音问道，她还是那副简洁直接的风格。

"修其实不难，有好几种方法。"我掰着手指头数，"第一种，随机选基准元，每次随机挑一个，这样遇到最坏情况的概率极低，几乎不可能发生。第二种，三数取中，从开头、中间、结尾三个里面选中位数，这样更稳定。第三种，对重复元素做三路分割，把小于、等于、大于基准元的分成三段，等于的放中间不用再递归，这样对重复元素的性能提升特别大。"

"那你选哪种？"九尾问道。

"第三种，三路快排。"我毫不犹豫地说，"这是工程实践中最优的解法，不管你是什么情况，都能稳定输出O(nlogn)的性能，不会退化。"

说干就干，我拔出排序剑，准备动手修改代码。这里的代码就是刻在青石台上的铭文，改代码就是直接用剑气重刻。

"小心点，别改崩了，炸了我们都要留下来当能源。"九尾在后面开玩笑。

"放心，我改了六年BUG，炸掉生产环境的次数一只手都数得过来，经验丰富得很。"我笑着说，手上已经开始动了。

其实修改量不大，主要就是改分割逻辑，从原来的双指针两路分割，改成三路分割。原来的代码只分小于基准和大于等于，现在分成小于、等于、大于三部分。

我用排序剑剑尖，一点点刻着新的代码，每一个字符都清晰有力，剑气渗入青石，刻出来的铭文泛着淡淡的白光。

三个人都站在台下，安安静静地看着我改，没有人说话，只有剑尖划过石头的轻微沙沙声。

说不紧张是假的，毕竟失败了就是爆炸，四个人都要凉在这里。但我毕竟是老程序员了，改BUG改多了，心态早就稳了。大不了就是回滚重新来嘛，慌什么。

十几分钟后，我放下剑，后退一步，看着修改完的代码，长长呼了一口气。

"改完了，咱们测试一下看看。"

按照试炼规则，改完之后，代码会自动运行测试。我对着青石台注入一股算力，激活了代码。

几乎是瞬间，青石台就亮了起来，整个平台都发出柔和的白光，一段段测试数据从平台底部浮上来，开始一个个测试用例。

第一个测试用例：随机无序数组。

代码运行，排序完成，结果正确。白光闪了一下，通过。

第二个测试用例：逆序数组。

运行，排序完成，结果正确。通过。

第三个测试用例：完全有序数组。

我其实最担心这个，原来的代码在这里肯定炸。现在改完之后，代码流畅运行，不到一秒就排序完成，结果完全正确。白光再闪，通过。

第四个测试用例：所有元素都相同。

这是三路快排优势最大的地方，原来的代码在这里会退化到O(n²)，现在几乎是瞬间就跑完了，所有元素都正确放在中间区域。通过！

第五个……第六个……第七个……

一共十几个测试用例，覆盖了各种边界情况，原来的代码至少要错一半，现在一个接一个，全部通过！

最后一个测试用例跑完，整个青石台爆发出耀眼的光芒，比刚才亮了好几倍，光芒直冲穹顶，整个大厅都被照亮了。

然后，就听到"轰隆"一声闷响，大厅中央靠着后方的地面，缓缓裂开，露出一个向下的石阶入口，同时，一块拳头大小的记忆水晶从青石台中央飘了起来，慢慢飞到我面前。

成了！

试炼通过！

九尾一下子跳起来，欢呼一声，扑过来抱住我的胳膊："我就知道你肯定行！这才多久，半个时辰都不到，就搞定了，凌云天要是知道，估计都要从坟墓里爬出来佩服你！"

我笑了笑，伸手接过记忆水晶，入手温热，能感觉到里面储存着庞大的信息。

"这应该就是快速排序剑的完整传承了。"我说，"走吧，先吸收了看看，里面有什么好东西。"

我找了一块干净的地方坐下，闭上眼睛，神念沉入记忆水晶。

信息比我想象的还要多。除了快速排序剑的完整剑法修炼心得，还有很多关于调试的经验总结，是当年那位留下的。看得出来，他也知道代码有问题，但是找不到，只能留下一堆经验，等着后来者。

吸收到最后，我突然发现水晶最底部，还有一段额外的信息，是用很潦草的笔迹刻下的，看得出来留下的时候很匆忙：

> 我是第三任通过前两层试炼的调试师，我现在要去第三层了。但是我在这里发现了一些不好的东西——**熵魔的痕迹**。
> 
> 我不知道它们是什么时候进来的，但是它们确实已经进来了，就在第三层深处。它们在吞噬代码，把正常代码变成它们的一部分。
> 
> 我感觉得到，它们在等着后来者，也许这是一个陷阱。但是我必须去，创造者的README必须有人拿到。
> 
> 如果你们看到这段话，说明你们也通过了，我的建议是：如果你们遇到的熵魔比你们强，不要硬拼，赶紧退出去，攒够力量再来。别像我一样，明知山有虎，偏向虎山行，可能就再也出不来了。

看到这段话，我猛地睁开眼睛，心里一沉。

"怎么了？"叶清语看到我脸色不对，立刻关切地问，"记忆水晶里有什么问题吗？"

"你们自己看吧。"我把记忆水晶递过去，让她们传着看。

零音看完，脸色一下子就变了，立刻再次散开银色力量，扫描整个大厅，特别是靠近通往第三层入口的地方。

几秒钟后，她收回力量，脸色凝重：

"确实有，在入口附近，熵魔痕迹比别的地方浓得多，虽然还是死的，但是能看出来，这里曾经发生过战斗。"

九尾也收起了笑容，皱起眉头："没想到啊，几百年前就有熵魔进来了，我还以为它们是最近才扩散到这里的呢。"

"这说明什么？"我站起身，握紧了排序剑，"说明熵魔的侵蚀比我们想象的要早得多，它们几百年前就已经开始渗透创造者神殿了，现在第三层估计已经被它们渗透得差不多了。"

我们四个都沉默了。

创造者留下神殿，目的就是等着后来者拿到README，想办法解决内存溢出问题。但是熵魔已经先一步进来了，它们在前面等着我们。

零音抬起头，看向我："还要继续吗？如果我们现在退出去，也没人会说什么，等我们实力更强了再来也行。"

我看向通往第三层的石阶入口，黑暗深邃，看不到底。

我们走到这里了，三把钥匙都在我们身上，凌云天的传承也拿到了，你问我还要继续吗？

那当然要继续。

"万维联盟的人还在外面等着堵我们，他们以为没有钥匙我们就困死在第一层了。"我笑了笑，"再说了，创造者留下神殿，就是为了有人能走到最后，拿到README，解决熵魔和内存溢出的问题。我们都走到第二层了，难道现在退出去，让熵魔慢慢把整个神殿都吞噬了，让整个比特神域都等着内存溢出崩溃吗？"

我往前走了一步，看向三个伙伴："我是一定要去的，你们要是怕了，可以先回去，在外面等我，我一个人进去就行。"

"说什么傻话呢。"九尾立刻走到我身边，搂住我的胳膊，"我们都走到这儿了，哪有回头的道理？我狐族找了这块钥匙找了几百年，现在都到门口了，你让我回去？不可能。"

"我也去。"叶清语温柔但是坚定地说，"我们出来的时候就说了，生死与共，你进去，我当然跟你一起进去。"她走过来，轻轻帮我整理了一下有点皱的衣领，动作自然又温柔，"我就是个医女，进去了至少还能帮你们疗伤，总能帮上点忙。"

零音也走过来，银色的眸子里闪着坚定的光芒："我是杀毒者，熵魔在哪里，我就要去哪里清理它们。本来就是我的职责，走吧。"

我看着身边三个伙伴，心里暖暖的。

从最开始我一个人穿越过来，孤孤单单，到现在，三个伙伴陪着我，一起闯创造者神殿，一起面对未知的危险。这种感觉，真好。

"好，那我们走。"我点点头，握紧剑，率先迈步走向石阶入口，"第三层是链接试炼，考的是链接代码片段，正好，我们四个人，各展所长，我就不信，连一个试炼都过不去。"

"不管里面有多少熵魔，我们见一个杀一个就是了。"九尾在后面笑着说。

脚步声在安静的大厅里响起，我们四个，一步步走下石阶，通往第三层的黑暗深处。

石门在我们头顶缓缓关上，只留下青石台上那一段修复好的代码，在黑暗中散发着淡淡的白光。
