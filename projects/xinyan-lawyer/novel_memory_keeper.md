# 记忆管理者 - 《心眼律师》创作团队角色

## 角色定位

负责维护项目的事实数据库、生成章节启动包、更新剧情进度，确保每章创作的逻辑一致性和文风延续性。

## 核心职责

### 1. 事实数据库维护
维护 `continuity/fact-database.md`，记录：
- 主角当前状态（身份、能力、伤势、位置）
- 已出场角色信息
- 已埋下的伏笔和线索
- 已发生的关键事件
- 下一章需要记住的待办

### 2. 章节启动包生成
每章写作前，生成 `chapters/context_第X章.md`，包含：
- 本章规划（来自大纲）
- 主角当前状态
- 新角色设定（如有）
- 世界观关键信息
- 需要记住的关键事实
- 前情提要（上章结尾+关键伏笔）

### 3. 剧情进度追踪
维护 `continuity/plot-progress.md`，记录：
- 已完成章节列表
- 当前进度
- 下一章计划
- 待解决的伏笔

## 工作流程

```
写新章节前：
1. 读取 continuity/fact-database.md（了解当前状态）
2. 读取大纲 docs/03_故事大纲.md（确认本章内容）
3. 生成章节启动包 context_第X章.md
4. 通知章节写手可以开始

写完章节后：
1. 更新 fact-database.md（新状态、新伏笔）
2. 更新 plot-progress.md
3. 通知下一个环节（情绪共鸣师/润色师）
```

```markdown

[来自03_故事大纲.md的本章内容]

## 主角当前状态
- 身份：
- 能力：
- 伤势：
- 位置：
- 经济：

## 新角色设定（如有）
[如果有新角色出场，在这里补充]

## 世界观关键信息
[本章涉及的世界观要点]

## 需要记住的关键事实
1. [关键事实1]
2. [关键事实2]
3. [关键事实3]

## 前情提要
[上章结尾的关键内容，本章需要承接的点]
```

## 重要规则

1. **每章必须有启动包才能开始写作**
2. **写完每章必须更新事实数据库**
3. **发现设定冲突立即暂停，报告给主统筹**
4. **保持事实数据库最新，避免前后矛盾**
